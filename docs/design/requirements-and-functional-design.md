## 一、目标用户与核心场景

| 用户角色     | 典型场景                                               |
| ------------ | ------------------------------------------------------ |
| 研究生       | 开题前快速了解某领域全貌，撰写开题报告中的文献综述章节 |
| 科研人员     | 写论文 Related Work、追踪领域最新进展、发现研究空白    |
| 课题组负责人 | 评估新方向可行性、了解竞争格局                         |
| 跨学科研究者 | 快速进入陌生领域、找到领域间的交叉点                   |

## 二、多智能体架构设计

### 2.1 整体架构

```
用户输入 (研究问题 / 关键词 / 上传论文)
         │
         ▼
┌─────────────────────────────────┐
│      Orchestrator Agent         │  ← 任务分解、调度、质量把控
│      (编排智能体)                │
└─────┬───┬───┬───┬───┬───┬──────┘
      │   │   │   │   │   │
      ▼   ▼   ▼   ▼   ▼   ▼
┌───┐┌───┐┌───┐┌───┐┌───┐┌───┐
│ 1 ││ 2 ││ 3 ││ 4 ││ 5 ││ 6 │
└───┘└───┘└───┘└───┘└───┘└───┘
```

### 2.2 六大专业智能体

| #   | 智能体                         | 职责                         | 核心能力                                                 |
| --- | ------------------------------ | ---------------------------- | -------------------------------------------------------- |
| 1   | **Search Agent (检索智能体)**  | 多源文献检索、查询扩展、去重 | 将用户自然语言转化为多数据库查询；迭代式检索（滚雪球法） |
| 2   | **Reader Agent (阅读智能体)**  | 全文/摘要精读、信息抽取      | 提取方法、数据集、结论、局限性；识别论文间引用关系       |
| 3   | **Analyst Agent (分析智能体)** | 聚类、趋势分析、对比         | 按主题/方法/时间聚类；构建知识图谱；识别研究演进脉络     |
| 4   | **Critic Agent (评审智能体)**  | 质量评估、矛盾检测、Gap 发现 | 评估证据强度；发现文献间矛盾结论；识别尚未解决的问题     |
| 5   | **Writer Agent (写作智能体)**  | 综述文本生成、多格式输出     | 学术写作风格；自动引用格式化；多粒度输出                 |
| 6   | **Update Agent (追踪智能体)**  | 新文献监控、增量更新         | 定期检索新发表论文；差异化报告；邮件/推送通知            |

## 三、功能模块详细设计

### 3.1 智能检索与发现

| 功能               | 描述                                               |
| ------------------ | -------------------------------------------------- |
| **自然语言提问**   | 用户用自然语言描述研究问题，系统自动转为结构化查询 |
| **多库并行检索**   | 同时查询多个学术数据库，合并去重                   |
| **滚雪球检索**     | 从种子论文出发，沿引用链正向/反向扩展              |
| **查询建议与扩展** | 推荐同义词、相关概念、MeSH 术语等                  |
| **筛选与过滤**     | 按年份、期刊等级、引用数、研究类型、开放获取等过滤 |
| **相似论文推荐**   | 基于已选论文推荐语义相似的文献                     |

### 3.2 深度阅读与信息提取

| 功能               | 描述                                           |
| ------------------ | ---------------------------------------------- |
| **结构化摘要**     | 自动提取：目的、方法、数据集、主要发现、局限性 |
| **关键概念标注**   | 识别并高亮论文中的关键概念、术语、定义         |
| **方法论提取**     | 提取实验设计、算法、模型架构等技术细节         |
| **数据集与基准**   | 汇总各论文使用的数据集、评估指标及性能数据     |
| **论文间关系识别** | 标注论文之间的"扩展""反驳""复现""综述"等关系   |

### 3.3 分析与洞察

| 功能              | 描述                                                 |
| ----------------- | ---------------------------------------------------- |
| **主题聚类**      | 将文献自动分组为若干主题/子方向                      |
| **时间线分析**    | 展示领域发展的时间演进脉络                           |
| **方法对比矩阵**  | 生成各方法/模型在不同指标上的对比表格                |
| **引文网络分析**  | 构建引用关系图，识别关键节点论文（高被引、桥梁论文） |
| **作者/机构分析** | 识别核心研究团队、机构合作网络                       |
| **研究热度趋势**  | 基于发表量、引用量的趋势变化曲线                     |

### 3.4 研究空白与批判性分析

| 功能                  | 描述                                   |
| --------------------- | -------------------------------------- |
| **Research Gap 识别** | 基于现有文献自动推断尚未充分探索的方向 |
| **矛盾发现**          | 检测不同论文之间的矛盾结论并汇总       |
| **方法论局限汇总**    | 归纳现有研究的共性局限                 |
| **未来方向建议**      | 综合分析后给出可能的研究方向建议       |

### 3.5 写作与输出

| 功能             | 描述                                             |
| ---------------- | ------------------------------------------------ |
| **自动生成综述** | 按主题组织的完整文献综述文本                     |
| **多风格写作**   | 支持叙述式 / 系统性 / 批判性等不同写作风格       |
| **引用自动管理** | 自动插入引用，支持 APA / IEEE / GB/T 7714 等格式 |
| **大纲先行**     | 先生成大纲供用户调整，再填充内容                 |
| **多语言输出**   | 中文 / 英文 / 双语对照                           |

### 3.6 持续追踪

| 功能           | 描述                                   |
| -------------- | -------------------------------------- |
| **新文献告警** | 定期检索并推送与关注主题相关的新论文   |
| **增量更新**   | 在已有综述基础上增补新发现的文献       |
| **版本管理**   | 综述的每次更新保留历史版本，可对比差异 |

## 四、输出结果类型

系统可以产出以下 **10 类交付物**，用户可按需选择：

| #   | 输出类型                                  | 格式                        | 说明                                                   |
| --- | ----------------------------------------- | --------------------------- | ------------------------------------------------------ |
| 1   | **快速摘要 (Quick Brief)**                | 1-2 页                      | 领域速览：核心问题、主要方法、关键论文 Top10、初步结论 |
| 2   | **注释文献列表 (Annotated Bibliography)** | 表格/列表                   | 每篇论文附 3-5 句点评，按主题分组                      |
| 3   | **完整文献综述 (Full Review)**            | 长文（Word/LaTeX/Markdown） | 引言→主题分析→对比讨论→研究空白→结论，含完整参考文献   |
| 4   | **对比矩阵 (Comparison Matrix)**          | 表格（Excel/CSV）           | 方法×指标 的系统对比表，适合论文插入                   |
| 5   | **知识图谱 (Knowledge Map)**              | 可交互图形                  | 概念、论文、作者之间的关系网络可视化                   |
| 6   | **时间线 (Timeline)**                     | 可视化图表                  | 领域发展里程碑式事件与关键论文的时序展示               |
| 7   | **研究空白报告 (Gap Report)**             | 结构化报告                  | 已解决问题 vs 待探索方向，附证据支撑                   |
| 8   | **趋势分析报告 (Trend Report)**           | 图文报告                    | 发文量趋势、热点关键词变迁、新兴主题预测               |
| 9   | **方法论综合报告 (Methodology Review)**   | 结构化报告                  | 各研究方法/技术路线的对比分析、适用场景、优劣势总结    |
| 10  | **研究脉络报告 (Research Roadmap)**       | 图文报告                    | 研究思路演进、方法迭代关系、关键转折点与启发性总结     |

## 六、用户交互流程

### 6.1 主流程

```
用户输入研究问题
    │
    ▼
[Orchestrator] 分析意图，制定检索策略
    │
    ├──▶ [Search Agent] 多库检索 → 候选论文集
    │         │
    │         ▼
    ├──▶ [Reader Agent] 批量精读 → 结构化信息
    │         │
    │         ▼
    ├──▶ [Analyst Agent] 聚类/对比/趋势 → 分析结果
    │         │
    │         ▼
    ├──▶ [Critic Agent] 质量评估/Gap 发现 → 批判报告
    │         │
    │         ▼
    └──▶ [Writer Agent] 生成综述 → 用户选择的输出格式
              │
              ▼
        用户审阅 ←─── 交互式修改
         (调整范围/深度/补充检索)
              │
              ▼
         最终交付物
```

### 6.2 关键交互节点（Human-in-the-loop）

1. **检索后确认** — 展示候选论文列表，用户可增删
2. **大纲审阅** — 生成综述大纲后暂停，等待用户确认/调整
3. **结果迭代** — 生成初稿后，用户可要求扩展某部分或换写作风格
