## 一、产品定位与交付形态

### 1.1 产品定位

**一句话描述**: 面向科研人员的 AI 驱动文献综述助手 — 输入研究问题，自动完成检索、阅读、分析、写作全流程，交付可直接使用的学术综述。

**核心价值**: 将传统需要数天至数周的文献综述工作，压缩至数分钟至数小时的人机协作过程。

### 1.2 交付形态

产品以 **Web 应用** 为主要交付形态，辅以命令行接口供高级用户和自动化场景使用：

| 交付形态            | 目标用户          | 说明                                             |
| ------------------- | ----------------- | ------------------------------------------------ |
| **Web 应用 (主要)** | 所有用户          | 浏览器访问，无需安装，全功能图形化界面           |
| **CLI 工具 (辅助)** | 开发者 / 高级用户 | 命令行调用，适合批量任务、脚本集成、CI/CD 流水线 |
| **API 服务 (开放)** | 第三方集成        | RESTful API，允许其他工具/平台接入综述能力       |

### 1.3 访问方式

| 方式          | 说明                                                         |
| ------------- | ------------------------------------------------------------ |
| **私有部署**  | Docker Compose 一键部署到个人服务器/实验室内网，数据完全自控 |
| **云端 SaaS** | 未来可选，注册即用，按用量计费                               |

## 三、用户界面总体设计

### 3.1 设计原则

| 原则              | 说明                                                             |
| ----------------- | ---------------------------------------------------------------- |
| **对话驱动**      | 以自然语言对话为主要交互方式，降低学习成本                       |
| **渐进披露**      | 默认简洁，高级选项按需展开，不让用户被复杂参数淹没               |
| **过程可见**      | Agent 工作过程实时可见（正在检索→正在阅读→正在分析），增强信任感 |
| **Human-in-loop** | 关键节点暂停等待用户确认，而非全自动黑盒                         |
| **结果可操作**    | 输出结果可直接编辑、导出、引用，无需二次加工                     |

### 3.2 整体布局


## 五、典型用户使用流程

### 5.1 场景一：研究生开题综述

```
步骤 1: 新建项目
  └─ 输入: "深度学习在医学影像分割中的应用，侧重 Transformer 架构"

步骤 2: 确认检索范围 (Human-in-loop #1)
  └─ 系统展示 58 篇候选论文
  └─ 用户: 去掉 2020 年之前的，保留 42 篇

步骤 3: 等待精读与分析
  └─ 状态栏: "Reader Agent 精读中 (18/42)..."
  └─ 约 3-5 分钟后完成

步骤 4: 查看分析看板
  └─ 浏览主题聚类、方法对比矩阵、研究趋势
  └─ 用户发现某个子方向论文不足，补充指令: "增加 SAM 模型的相关论文"

步骤 5: 确认综述大纲 (Human-in-loop #2)
  └─ 系统生成 6 章大纲
  └─ 用户: 合并第 3、4 章，增加一节 "临床应用案例"

步骤 6: 生成综述全文
  └─ Writer Agent 按确认后的大纲生成完整综述
  └─ 用户在编辑器中微调表述

步骤 7: 导出
  └─ 选择 Word 格式 + GB/T 7714 引用格式
  └─ 下载文件，直接放入开题报告
```

### 5.2 场景二：科研人员快速调研

```
步骤 1: 输入: "multi-agent LLM 最新进展 2024-2025"
步骤 2: 选择输出类型: Quick Brief
步骤 3: 3 分钟后获得 2 页领域速览
  └─ 包含: Top 10 关键论文、3 个主要方向、核心结论
步骤 4: 对某个方向感兴趣，追问: "展开第2个方向的详细综述"
步骤 5: 获得该子方向的 Full Review
```

### 5.3 场景三：从已有论文出发

```
步骤 1: 上传 3 篇 PDF 论文作为种子文献
步骤 2: 输入: "找到这些论文的相关工作，做一个系统性综述"
步骤 3: 系统通过引用链扩展到 35 篇相关论文
步骤 4: 查看分析看板中的引文网络图
步骤 5: 生成综述，导出为 LaTeX + IEEE 引用格式
```

## 六、输出交付物的呈现方式

| 输出类型               | 界面呈现方式                                      | 支持的导出格式                |
| ---------------------- | ------------------------------------------------- | ----------------------------- |
| **Quick Brief**        | 卡片式摘要，一屏可览                              | Markdown / PDF                |
| **Annotated Bib**      | 可排序表格，每行展开显示详细注释                  | Excel / CSV / BibTeX          |
| **Full Review**        | 富文本编辑器，支持在线修改                        | Word / LaTeX / Markdown / PDF |
| **Comparison Matrix**  | 交互式表格，支持排序、高亮、筛选                  | Excel / CSV / 图片            |
| **Knowledge Map**      | 可交互的力导向图 / 树形图，节点可点击查看论文详情 | SVG / PNG / HTML              |
| **Timeline**           | 横向时间轴，里程碑标注，鼠标悬停显示详情          | SVG / PNG / HTML              |
| **Gap Report**         | 结构化卡片：已解决 vs 待探索，证据链接            | Markdown / PDF                |
| **Trend Report**       | 图表 + 文字解读的组合页面                         | PDF / HTML                    |
| **Methodology Review** | 方法对比卡片 + 适用场景矩阵，支持在线编辑         | Word / Markdown / PDF         |
| **Research Roadmap**   | 演进流程图 + 关键节点文字解读，交互式展示         | Markdown / PDF / HTML         |
